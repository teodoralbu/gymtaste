## Spacing Scale

This project uses a **4-point grid** (not the 8-point standard set). All spacing tokens are multiples of 4. The extended set includes 12px and 20px which are established project tokens defined as CSS custom properties in `globals.css`.

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 8px (`--space-xs`) | Compact element spacing, icon gaps |
| sm | 12px (`--space-sm`) | Tight padding inside compact elements |
| md | 16px (`--space-md`) | Screen side padding, default element spacing |
| lg | 20px (`--space-lg`) | Comfortable internal padding |
| xl | 24px (`--space-xl`) | Section gaps |
| 2xl | 32px (`--space-2xl`) | Major section breaks |

Extended set: {4, 8, 12, 16, 20, 24, 32, 48, 64}

Exceptions: Skeleton loading blocks use exact pixel heights to match the content they replace (see Loading States section). The 4px value is available for fine adjustments (e.g., outline-offset) but has no named token.

## Typography

Existing project type scale (from globals.css and component inspection). Uses exactly 2 weights: 400 (regular) for body text and 700 (bold) for all emphasis roles.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label / Caption | 13px | 700 (bold) | 1.4 |
| Section Title | 18px | 700 (bold) | 1.2 |
| Page Heading | 20px | 700 (bold) | 1.2 |

Source: globals.css `.m-section-title` (updated from 17px/800 to 18px/700), `.btn` (14px/700), `.m-segment-tab` (13px, updated from 600 to 700), `.input` (16px), body inherited.

Font size rationale: The 4 sizes (13, 16, 18, 20) provide clear hierarchical steps. The Section Title was adjusted from 17px to 18px to create a perceptible 2px step from 16px Body rather than an imperceptible 1px difference.

Weight consolidation: Previous spec used 4 weights (400, 600, 700, 800). Collapsed to 2 weights: 400 for body text, 700 for all emphasis (labels, titles, headings). During implementation, update `.m-section-title` from `font-weight: 800` to `font-weight: 700` and `.m-segment-tab` from `font-weight: 600` to `font-weight: 700`.

No new type styles introduced in this phase. Empty state and loading skeleton text must use these existing roles only.

## Color

Existing token system (from globals.css `:root` and `[data-theme="light"]`):

| Role | Dark Value | Light Value | Usage |
|------|------------|-------------|-------|
| Dominant (60%) | `--bg: #0D0F14` | `--bg: #F2F3F5` | Page background |
| Secondary (30%) | `--bg-card: #161B22` | `--bg-card: #FFFFFF` | Cards, elevated surfaces, skeleton base |
| Accent (10%) | `--accent: #00B4FF` | `--accent: #0095D9` | Focus rings, primary CTA, active tab, accent links |
| Destructive | `--red: #FF4444` | `--red: #E03030` | Error messages, destructive actions only |

Accent reserved for: focus-visible outline, primary action buttons (`.btn-primary`), active segment tab indicator, input focus border glow, range slider thumb.

### WCAG AA Contrast Fixes Required

These color adjustments are part of the UX-03 requirement. The executor must verify and fix:

| Pair | Current | Required Action |
|------|---------|-----------------|
| `--text-dim` on `--bg-card` (dark) | `#556070` on `#161B22` | Lighten to minimum `#6B7A90` (4.5:1 ratio) if used for readable text |
| `--text-faint` on `--bg` (dark) | `#374050` on `#0D0F14` | Acceptable ONLY for placeholder/disabled text. Must NOT be used for readable content |
| `--text-faint` on `--bg` (light) | `#B0B8C8` on `#F2F3F5` | Acceptable ONLY for placeholder/disabled text. Must NOT be used for readable content |
| `--text-dim` on `--bg-card` (light) | `#7A8090` on `#FFFFFF` | Verify passes 4.5:1. If not, darken to `#6A7080` |

Rules:
- `--text-faint` is restricted to placeholder text and disabled UI only. No readable content may use it.
- `--text-dim` must pass 4.5:1 against its background in both themes. Adjust the token value if it does not.
- `--text-muted` is the minimum color for secondary readable text. No adjustments expected (already passes).

|----------|-------------------|
| Home page (`/`) | 32px heading skeleton (width 200px), row of 4 stat card skeletons (64px tall), 3 feed card skeletons (140px tall) |
| Flavor detail (`/flavors/[slug]`) | 24px heading skeleton (width 240px), 200px image placeholder skeleton, 2 text line skeletons (16px tall) |

All skeletons use the existing `.skeleton` CSS class (shimmer animation from globals.css). Skeleton containers use `border-radius: var(--radius-md)` for blocks and `border-radius: 50%` for avatar circles. Skeleton dimensions must match actual content dimensions to prevent layout shift.

### Error States

| Context | Message | Recovery |
|---------|---------|----------|
| Supabase query failure (page level) | Something went wrong | Try refreshing the page. If the problem continues, check back later. |
| Supabase query failure (component level) | Could not load [entity] | Fail silently to `[]` or `null` with `console.error`. Do not show error UI for partial failures. |

## Accessibility Contract

### Focus Visibility (UX-01)

| Element Type | Required Treatment |
|--------------|--------------------|
| Native `<button>`, `<a>`, `<input>` | Global `:focus-visible` rule applies automatically (2px solid `var(--accent)`, 2px offset) |
| `<div>` used as button (e.g., AvatarUpload) | Add `role="button"`, `tabIndex={0}`, `onKeyDown` for Enter/Space, `aria-label` |
| `<div>` used as link (e.g., card click handlers) | Replace with `<Link>` or `<a>` wrapping. If not possible, add `role="link"`, `tabIndex={0}`, keyboard handler |
| Range slider thumb | Inherits focus from `<input type="range">` -- verify `:focus-visible` outline is visible on the track element |
| `.input` class elements | Already handled: `outline: none` compensated by `border-color: var(--accent)` + `box-shadow` on `:focus` |

### Image Alt Text (UX-02)

| Image Type | Alt Text Pattern |
|------------|-----------------|
| Product image | `"{product name} by {brand name}"` |
| Flavor image | `"{flavor name} - {product name}"` |
| User avatar | `"{username}'s avatar"` |
| Decorative SVG icons | `aria-hidden="true"` on the `<svg>` element |
| Brand logo | `"{brand name} logo"` |

### Contrast (UX-03)

See Color section above for specific pairs and required adjustments.

Verification method: Use browser DevTools contrast checker on each `--text-dim` and `--text-faint` usage. Do not rely on manual hex math.
