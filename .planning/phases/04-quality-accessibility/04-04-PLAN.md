  <action>
Audit every `<img>`, `<NextImage>` (aliased as `Image`), and `<svg>` tag in all 6 component files. Apply alt text per the UI-SPEC patterns:

**Alt text rules:**
- Product images: `alt={`${product.name} by ${brand.name}`}` — use the variable names available in scope
- Flavor images: `alt={`${flavor.name} - ${product.name}`}` — use variable names in scope
- User avatars: `alt={`${username}'s avatar`}` — use username variable in scope
- Brand logos: `alt={`${brand.name} logo`}` — use brand.name variable
- Decorative SVG icons (hamburger, close, arrows, stars): add `aria-hidden="true"` to the `<svg>` element

**For each file:**
1. Find all image elements (`<img`, `<Image`, `<NextImage`)
2. Check if `alt` prop exists and is descriptive
3. If `alt=""` (empty) on an informational image, add descriptive text using the patterns above
4. If `alt` is missing entirely on `<img>`, add it
5. NextImage already requires `alt` prop (TypeScript enforces), but check that the value is descriptive not just `alt=""`
6. Find all `<svg>` elements that are decorative icons and add `aria-hidden="true"` if not present

**Common patterns to look for and fix:**
- `alt=""` on product/flavor images -> change to descriptive text using data variable names
- `alt="avatar"` or `alt="profile"` -> change to `alt={`${username}'s avatar`}`
- Missing `aria-hidden="true"` on icon SVGs (search, heart, comment, star icons)

Do NOT modify SVGs that convey meaning (e.g., a star rating that is the only indicator of rating value). Those need `aria-label` instead.
  </action>
