## Spacing Scale

Declared values (from existing `globals.css` tokens -- no changes for this phase):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 8px (`--space-xs`) | Compact element spacing |
| sm | 12px (`--space-sm`) | Small gaps |
| md | 16px (`--space-md`) | Default element spacing, page horizontal padding |
| lg | 20px (`--space-lg`) | Section internal padding |
| xl | 24px (`--space-xl`) | Section margin-bottom |
| 2xl | 32px (`--space-2xl`) | Major section breaks |

**Standard-set exemptions:** The following values fall outside the standard spacing set {4, 8, 16, 24, 32, 48, 64} but are pre-existing inherited tokens defined in `globals.css` prior to this phase. This phase does NOT introduce or modify these values; they are documented here for completeness only and are therefore exempt from the standard-set constraint:

- `--space-sm` (12px) -- inherited from `globals.css`, used project-wide for small gaps
- `--space-lg` (20px) -- inherited from `globals.css`, used project-wide for section internal padding
- Hero text wrapper padding (28px top/bottom) -- inherited from existing hero card padding pattern in `page.tsx`

Exceptions: Hero image padding is 0px (edge-to-edge within card). Hero text content wrapper uses 28px top/bottom, 20px left/right (matching existing hero card padding pattern).

## Color

Existing 3-theme system (blue/light/black) -- no new colors for this phase.

| Role | Value (blue theme) | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--bg)` #080B12 | Page background |
| Secondary (30%) | `var(--bg-card)` #0F1420 | Cards, hero card, nav |
| Accent (10%) | `var(--accent)` #3D8EFF | Primary CTA buttons, focus rings, active nav icon |
| Destructive | `var(--red)` #FF4444 | Not used in this phase |

Accent reserved for: Primary CTA links on hero card ("Get Started", "Sign Up"), active navigation icon highlight, focus-visible outlines.

### Phase-Specific Color Rules

- **FIX-02**: Replace `text-white` with `var(--text)` on the email display span in signup success screen. This ensures readability across all three themes.
- **Hero image placeholder**: If using a gradient placeholder instead of an image file, use `linear-gradient(135deg, var(--bg-elevated), var(--bg-hover))` to stay theme-consistent.

## Visual Specifications

### FIX-04: Bottom Navigation (4 Tabs)

```
Layout: Home | [Rate button] | Top | Profile
Each tab: flex: 1 (25% width)
Rate button: absolutely positioned, centered at 50% of container
Nav height: 64px (--nav-height, unchanged)
Rate float button: 54px (--nav-float-size, unchanged)
```

No visual change to individual tab icons or labels. The Rate button remains centered because it uses absolute positioning with `left: 50%; transform: translateX(-50%)`.

### FIX-05: Hero Image Card

```
Card container:
  - border-radius: var(--radius-lg) (16px)
  - background: var(--bg-card)
  - border: 1px solid var(--border)
  - overflow: hidden (REQUIRED -- clips image to rounded corners)
  - padding: 0 (moved to inner wrapper)

Image:
  - Component: next/image
  - src: "/hero-placeholder.jpg" (swappable placeholder)
  - alt: "Pre-workout supplement"
  - width: 800, height: 400 (aspect ratio hint)
  - style: width 100%, height auto, maxHeight 200px, objectFit cover, display block
  - priority: true (above fold)
  - Position: first child of card, before text content

Text wrapper (below image):
  - padding: 28px 20px
  - Contains: h1 tagline, description, CTA links
```

### FIX-02: Email Text Fix

```
Before: <span className="text-white font-medium">{form.email}</span>
After:  <span style={{ color: 'var(--text)', fontWeight: 500 }}>{form.email}</span>
```

Uses inline style with CSS variable to ensure readability on all three themes (blue, light, black).

## Accessibility Notes

- **FIX-02**: Fixes a WCAG AA contrast violation -- white text on light theme background is unreadable. Using `var(--text)` restores proper contrast ratios across all themes.
- **FIX-05**: Hero image must have descriptive `alt` text: "Pre-workout supplement". The `next/image` component handles this natively.
- **FIX-04**: Removing Browse tab does not remove access to `/products`, `/flavors`, `/brands` routes -- they remain accessible via in-page links. No navigation capability is lost.
